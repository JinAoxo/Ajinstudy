Tip list

HTML Tip :>

  1. 1. table > table의 경우 접근성 문제에서 오류가 발생할 가능성이 있기 때문에 쓰지 않는 편이 좋다(But. 그냥 쓰는 경우가 많다)
  2. 2. table 속성의 경우 table이 가지고 있는 속성을 잘 생각하고 쓰는것이 도움이 된다. => table은 데이터(DB)를 불러오는 경우 쓴다.
  3. 3. a태그의 주의점(>input) : a태그안에 input 태그는 쓰이면 안됩니다. ⇒ 접근성 위배되기 때문입니다.
  4. 4. lable태그의 주의점(>p) : lable 안에 span만들어갈 수 있지만, p 태그는 들어가면 안됩니다.
  5. 5. form + fieldset + legend ==⇒ 전송해야하는 내용이 있는경우 필수 접근성
  6. 6. pc 레이아웃을 만들때는 항상 container 또는 wrap 박스를 만들어 놓는것이 좋습니다. 또한 conainer 박스에는 width를 지정해 놓는 것이 좋습니다. 그래야 브라우저 화면이 축소되더라도 내부가 찌그러지지 않습니다.

CSS Tip :>

  1. 1. display : table; 주의점 : display: table 은 display: table-cell과 꼭 함께 쓰는것이 좋습니다. 테이블로 정의 해줬기 때문에 Cell도 필요하기 때문입니다.
  2. 2. display : flex; 주의점 : display: flex 안의 flex는 지양하는 것이 좋습니다. 상위 요소와 하위 요소가 충돌을 일으킬수 있기 때문입니다.
  3. 3. padding 속성의 특징: padding의 수직값은 상위 요소의 height 속성이 아닌 width 속성에 영향을 받습니다.
  4. 4. css를 ilne으로 html파일에 넣는것은 안하는것이 좋습니다.
  5. 5. position : absolute; 를 이용한 가운데 정렬 : left:0; right:0; margin-left:auto; margin-right:auto; width:적절히?; 5개의 속성을 사용해야 가운대 정렬이 가능합니다.
